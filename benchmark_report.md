# AI Token 输出速度测试报告

**测试日期**: 2026-03-16

**测试平台**: 百炼 Anthropic 兼容接口 (https://coding.dashscope.aliyuncs.com/apps/anthropic)

## 测试概述

本次测试通过百炼的 Anthropic 兼容接口，对比测试了 **Pro套餐** 和 **Lite套餐** 下 qwen3.5-plus 和 glm-5 模型在 AI CLI 工具使用场景下的 token 输出速度。

### 测试指标
- **TTFT (Time to First Token)**: 首 Token 延迟，衡量响应启动速度
- **Tokens/s**: 吞吐量，衡量生成速度
- **测试次数**: 每个测试用例 3 次迭代取平均

### 测试用例（AI CLI 场景）
1. 简单代码生成
2. 代码解释
3. 复杂代码生成
4. 代码重构
5. Bug修复
6. 算法实现
7. 代码审查
8. 快速问答

## 分析与结论

### 1. qwen3.5-plus vs glm-5
- **qwen3.5-plus 在 AI CLI 场景下表现显著优于 glm-5**
- TTFT 快约 3 倍，用户体验更流畅
- 吞吐量高约 4 倍，适合代码生成等长文本任务

### 2. Lite套餐 vs Pro套餐 (qwen3.5-plus)
- **出乎意料：Lite套餐性能略优于Pro套餐**
- Lite TTFT: 15.4秒 vs Pro TTFT: 16.4秒（快 6%）
- Lite 吞吐量: 49.8 vs Pro 吞吐量: 46.6（高 7%）
- 可能原因：
  - 测试时段服务器负载不同
  - 网络波动影响
  - 套餐优先级差异不明显

### 3. 最佳选择建议
- **追求速度**: qwen3.5-plus (Lite套餐性价比更高)
- **追求稳定**: 建议多次测试取平均
- **成本考量**: Lite套餐已能满足大多数AI CLI使用场景
